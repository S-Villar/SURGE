# Development & Modification Tracking

This folder contains documentation for **development status**, **modification tracking**, and **application-specific notes** that are not part of the core SURGE user-facing documentation.

## Purpose

- **Modification tracking** – summaries of unstaged changes, refactors, and migration notes
- **Status & reviews** – poster reviews, comparison with other tools, capability assessments
- **Potential features** – guides for features under consideration (e.g. multimodal diffusion)
- **Application-specific** – notes on how to obtain or integrate external code (e.g. M3DC1 Python support)
