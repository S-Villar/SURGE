Citing SURGE
============

The software is registered in `U.S. DOE OSTI / DOE Code
<https://www.osti.gov/doecode>`__ (Code ID `179819
<https://www.osti.gov/doecode/biblio/179819>`__) with DOI
`10.11578/dc.20260422.5 <https://doi.org/10.11578/dc.20260422.5>`__.
OSTI’s project page also exports APA, Chicago, **MLA** (excerpt below),
BibTeX, EndNote, and RIS under *Citation Formats*.

**MLA (matches OSTI’s export):**

Sanchez-Villar, Alvaro, Churchill, R. Michael, Jha, Shantenu, Churchill, R. Michael, and Jha, Shantenu. *SURGE - Surrogate Unified Robust Generation Engine.* Computer Software. `https://github.com/S-Villar/SURGE <https://github.com/S-Villar/SURGE>`__. USDOE Office of Science (SC), Fusion Energy Sciences (FES). 22 Apr. 2026. Web. doi:10.11578/dc.20260422.5.

The repository’s root ``CITATION.cff`` lists the three people above once
each (CFF disallows duplicate author entries). GitHub and BibTeX exports
use that list; the MLA line above is the one **published by OSTI** and
repeats Churchill and Jha in the “Project Member” part of the record.

The ``CITATION.cff`` also carries the DOI, the OSTI page URL, and
``preferred-citation`` for tooling. Version and URL should match the
release you used (see ``pyproject.toml`` and tags on
`GitHub <https://github.com/S-Villar/SURGE>`__).
